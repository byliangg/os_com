赛题背景
传统操作系统及文件系统多基于C语言构建，手动内存管理模式易引发缓冲区溢出、悬垂指针等缺陷，行业数据估计有60%-70% 的系统安全漏洞源于此。文件系统为保障数据一致性，普遍采用JBD2日志等崩溃恢复技术，但这类方案会产生碎片化元数据I/O、约束I/O顺序，形成 安全一致性与高性能”的核心矛盾。
基于Rust的星绽操作系统凭借创新性framekernel架构，将可信计算基（Trusted Computing Base）最小化，通过OSTD框架为文件系统提供非特权级安全运行环境，依托Rust所有权语义从根源规避内存安全风险，同时兼容Linux ABI，具备主流生态迁移基础。但星绽操作系统目前未支持EXT4等主流文件系统，在大数据、云原生等场景下的部署受到限制。

星绽的安全架构对文件系统提出“Rust原生实现、适配非特权运行”的新要求，也为性能优化提供契机——可依托架构特性简化传统安全校验流程，缓解一致性与性能的冲突。因此，在星绽上用Rust实现兼容POSIX接口、支持JBD2日志与Extent特性的EXT4，并探索性能优化技术，既能填补星绽生态空白，推动国产安全OS部署，也能为RustOS存储层设计提供可行的性能优化方案，兼具学术与产业价值。

赛题任务
任务内容：
研究星绽OS framekernel架构与OSTD非特权运行环境的技术特性，设计适配星绽架构的Rust版EXT4分层架构、文件系统核心功能，并遵循xfstest测试规范完成文件系统功能正确性验证，按梯度实现POSIX接口与EXT4核心特性；

开展面向RustOS/星绽架构的EXT4性能优化技术探索，结合Rust 安全编程语言特性以及星绽OS的内存管理、硬件交互特性，探索RustOS下的EXT4性能优化策略（如Extent块分配算法优化、元数据I/O调度优化、崩溃一致性协议优化等），验证优化策略的有效性并形成研究结论。

技术指标：
基于星绽实现的EXT4应实现POSIX文件系统操作接口，包括create, open, close, read, write, truncate, lseek, mkdir, rmdir, unlink, rename, stat等。
支持EXT4基本特性，包括日志功能(Journaling Block Device 2, JBD2)和Extent 连续块管理功能。
基于星绽实现的EXT4需在I/O吞吐及延迟上接近Linux发行版EXT4。
探索基于RustOS的文件系统性能优化技术。
赛题特征
测试环境：
硬件环境：服务器或个人PC
测试环境：基于QEMU虚拟机搭建标准化测试环境，分别部署Linux系统与星绽操作系统；存储设备统一通过 virtio-blk 标准化虚拟化接口实现管理与数据交互。
功能正确性：遵循 xfstest 开源文件系统测试套件的测试规范与用例要求，针对所实现的EXT4文件系统核心功能开展一致性测试，确保文件系统功能行为符合xfstest定义的正确性标准。
性能：使用FIO、filebench或其他测试工具评估文件系统基本性能，使用SQLite或其他应用测试文件系统在真实工作环境下的性能表现。

评审要点
一、实现完整度（40%）
基础档：实现EXT4基础文件和目录操作管理功能，支持单文件基本读写，遵循 xfstest 规范完成基础功能验证；
进阶档：实现EXT4基础日志操作管理功能（日志记录、简单崩溃恢复），支持单文件基本读写，完成崩溃恢复基础测试（模拟掉电后数据一致性验证）；
优秀档：实现日志刷盘、事务管理、全量崩溃恢复、多文件并发读写等进阶功能，完成多场景崩溃恢复测试、并发读写测试无数据错乱和丢失问题。
二、文档完整性（20%）
基础档：完成架构设计文档、用户手册、测试文档，文档内容完整、逻辑清晰；
进阶档：完成基础档所有文档，新增开源Rust EXT4差异化分析文档，文档包含技术对比与分析结论；
优秀档：完成进阶档所有文档，新增Rust与C版EXT4性能优化技术研究报告，报告包含量化测试数据、明确研究结论，符合学术型文档要求。
三、Demo质量（20%）
基础档：Demo 可正常运行，实现基础档功能，性能指标达到 Linux EXT4 的 70% 及以上；
进阶档：Demo 无功能 BUG，实现进阶档功能，性能指标达到Linux EXT4的80% 及以上，支持简单崩溃恢复；
优秀档：Demo运行稳定，实现优秀档功能，性能指标达到Linux EXT4 的 90% 及以上，所探索的性能优化技术实现≥5% 的性能提升，并发读写无数据问题。
四、创新性（20%）
基础档：完成星绽OS的EXT4 适配；
优秀档：探索1-2种面向RustOS/星绽架构的EXT4性能优化技术，优化策略合理、测试数据可信，形成可复用的RustOS文件系统优化方法，体现学术型研究的创新性。
